## 分支命名
- 格式：`feature/{username}_模块名` 或 `bugfix/{username}_模块名`
- 模块名用下划线连接，最多两个单词，如 `feature/domchen_layer_content`

## 发起 Github PR
- 先创建新分支并提交到本地仓库
- PR 标题：英语，格式同 commit 信息的要求
- PR 描述：中文详细说明
- 信息准备好后需经我确认，再推送并执行 `gh pr create`
